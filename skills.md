1. Skill: Flight Data Integration (데이터 수집)
목표: 공공데이터포털 API를 호출하여 특정 항공편의 기초 데이터를 확보함.

세부 기능:

항공기 출발/도착 예정 시간 및 실시간 상태(출발, 도착, 지연, 결항) 조회.

특정 항공기 편명(예: KE123)을 기준으로 한 실시간 위치 좌표(위도, 경도) 및 고도 데이터 수집.

카파시 원칙 적용:

Think Before Coding: API 응답의 JSON/XML 구조를 먼저 분석한 뒤 파싱 로직을 설계함.

Simplicity First: 복잡한 DB 구축 대신, 실시간 조회를 위한 경량화된 API 호출 로직을 유지함.

2. Skill: Progress & Path Analysis (진행 및 경로 분석)
목표: 수집된 좌표와 시간 데이터를 바탕으로 비행의 진행률을 계산함.

세부 기능:

출발지-목적지 거리 대비 현재 위치의 진행률(%) 계산.

남은 비행시간 및 예상 도착 시간(ETA) 산출.

카파시 원칙 적용:

Goal-Driven Execution: 비행 경로를 3D로 화려하게 그리는 것보다, "지금 어디쯤 왔는가"라는 핵심 질문에 답하는 로직에 집중함.

3. Skill: Environmental Context (환경 정보 결합)
목표: 도착지의 기상 정보를 결합하여 항공기 착륙 가능 여부 및 환경을 파악함.

세부 기능:

기상청 API 연동을 통한 도착 공항의 실시간 기상(강수, 풍속, 시정) 정보 매핑.

카파시 원칙 적용:

Surgical Changes: 기존 항공 데이터 로직을 건드리지 않고, 기상 정보만 별도의 모듈로 '외과수술적'으로 추가함.

4. Skill: Dashboard Visualization (직관적 시각화)
목표: 사용자(가족/지인)에게 '버스 정류장 안내판'처럼 직관적인 정보를 제공함.

세부 기능:

가독성을 최우선으로 한 상태 메시지 출력 (예: "30분 뒤 도착 예정", "기상 악화로 회항 중").

Streamlit 등을 활용한 단순하고 명확한 대시보드 UI 구현.

카파시 원칙 적용:

Simplicity First: 화려한 애니메이션을 배제하고 텍스트와 핵심 지표 중심의 UI 구성.
